Final decisions:
1. requires_human=true defaults to soft.
2. handoff_required=true defaults to hard.
3. escalation_mode="soft" or "ai_help" normalizes to soft.
4. escalation_mode="hard" or "human_takeover" normalizes to hard.
5. Hard can upgrade soft from webhook.
6. Soft cannot downgrade hard from webhook.
7. Existing escalated=true rows with no mode show legacy grey “Escalation”.
8. Takeover does NOT auto-resolve.
9. Soft guidance does NOT auto-resolve.
10. human_responder / created_by can be client_slug for v1.
11. Jr/AI layer is responsible for reading ai_muted and stopping automatic replies.
12. Hard reply in Phase 1 must be a placeholder/recording action only. It must not pretend the message was sent to the customer.
13. AI rewrite icon can be planned/disabled only. Do not call external AI APIs.

Add routes:
POST /:client/dashboard/api/escalations/:id/guidance
Body:
{
  "guidance": "...",
  "saveToYourInfo": true/false,
  "autoUseNextTime": true/false,
  "category": optional
}
Behavior:
- store human_guidance
- human_responder = client slug
- human_responded_at = NOW()
- set learn_from_resolution
- set ai_may_use_automatically
- if saveToYourInfo=true create learning_entries row
- status suggested by default, approved if autoUseNextTime=true
- update learning_status
- do NOT resolve escalation automatically
- return { ok: true }

POST /:client/dashboard/api/escalations/:id/takeover
Behavior:
- set escalation_mode='hard'
- set human_takeover_at=NOW()
- set ai_muted=true
- do NOT resolve
- return { ok: true }

POST /:client/dashboard/api/escalations/:id/handback
Behavior:
- set escalation_mode='soft'
- set ai_muted=false
- do NOT resolve
- return { ok: true }

POST /:client/dashboard/api/escalations/:id/mode
Body:
{ "mode": "soft" | "hard" }
Behavior:
- mode hard sets ai_muted=true and human_takeover_at if null
- mode soft sets ai_muted=false
- do NOT resolve
- return { ok: true }
