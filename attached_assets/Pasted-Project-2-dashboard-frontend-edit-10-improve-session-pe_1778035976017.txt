Issue:
Dashboard sessions expire too often.
The user is forced to log in repeatedly while using the dashboard.
This is unacceptable for a customer dashboard.

Task:
Audit and fix the frontend auth/session handling so the dashboard does not log out unless the backend token is truly invalid/expired or the user explicitly signs out.

Important:
The real token lifetime is controlled by the Python backend.
Do not invent backend refresh logic unless the API supports it.
But fix any frontend-side causes of unnecessary logout.

Required behavior:
1. Store the login token persistently in localStorage, not short-lived component state.
2. On page refresh, restore the token from localStorage.
3. Do not clear the token just because one non-auth API request fails with network/server error.
4. Only clear token and redirect to login on clear authentication failure:
   - HTTP 401
   - HTTP 403 if backend means invalid/expired token
5. If API is temporarily unreachable, show a connection/server error but keep the user logged in.
6. If token expires, show a clear message:
   “Session expired. Please sign in again.”
7. Sign out button should still clear token immediately.
8. Avoid login loops.
9. Avoid clearing token on page reload.
10. Keep existing login UI and password flow.
