Goal:
Escalations need a proper reply composer.

There are two escalation types:

1. Soft escalation
The operator writes guidance/reply to Marina.
Marina will use that guidance to reply to the customer.
The operator is NOT replying directly to the customer.

2. Hard escalation
The operator writes a reply directly to the customer.
The AI is muted / human takeover.
The reply is intended to go to the customer.

Both composer types must include an AI Editor inspired by the provided screenshot.

AI Editor actions:
- Translate
- Style
- Fix

Important:
The AI Editor improves the operator’s draft, but never sends automatically.
The operator must approve/send manually.

Use Refero/Repro:
Use real premium AI composer/editor references before designing.
References should include:
- Grammarly editor
- Intercom AI compose/rewrite UI
- Zendesk AI assist
- Front email composer
- Notion AI editor
- Apple writing tools style
- Slack message composer

If Refero/Repro is not connected, stop and report clearly before editing.

Frontend API behavior:
Create clean frontend API client functions, but do not build backend.

Suggested endpoints to call when Jr wires backend:

POST /api/{client}/dashboard/api/escalations/{conversationId}/guidance
For soft escalation.
Body:
{
  "message": "operator guidance text",
  "mode": "soft",
  "target": "marina"
}

POST /api/{client}/dashboard/api/escalations/{conversationId}/reply
For hard escalation.
Body:
{
  "message": "direct customer reply text",
  "mode": "hard",
  "target": "customer"
}

POST /api/{client}/dashboard/api/ai-editor
For AI Editor.
Body:
{
  "action": "translate" | "style" | "fix",
  "text": "draft text",
  "targetLanguage": "English/Dutch/Spanish/Papiamento/Swedish/Portuguese optional",
  "style": "professional/warmer/shorter/friendlier/direct optional",
  "context": {
    "conversationId": "...",
    "escalationMode": "soft" | "hard",
    "channel": "whatsapp/email/instagram/facebook/messenger/etc"
  }
}

Expected response:
{
  "text": "edited text"
}
