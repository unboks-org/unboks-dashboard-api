Issue:
Conversation rows currently show the channel badge inline with the message preview, e.g.:

WhatsApp · Let me flag this conversation...

This works but does not feel premium.
It creates visual noise, especially when the user is already inside the WhatsApp channel filter.
The section header also just says “WhatsApp” without helpful context.

Task:
Improve the UX and visual hierarchy of the conversation list rows and channel-filter context.

Goals:
- Make the inbox feel like a polished SaaS inbox.
- Make the source channel visible without clutter.
- Make channel-filter pages feel intentional.
- Do not change backend/API behavior.
- Do not change the actual filtering logic.

Required row layout:
1. First line:
   - customer/contact name on the left
   - timestamp and star on the right
2. Second line:
   - latest message preview
   - channel badge should be shown cleanly, not jammed into the sentence
3. Channel badge:
   - small pill or compact badge
   - icon/dot + label
   - visually separate from the message preview
   - should not dominate the row
4. If the active filter is a specific channel, e.g. WhatsApp:
   - hide the repeated WhatsApp badge in every row OR reduce it to a very subtle context label.
   - Preferred: hide channel badge when active channel filter equals the row channel.
5. If active view is Inbox/all:
   - show the channel badge, because rows can come from different sources.

Required page/list header:
When Inbox/all is selected:
- Title: Inbox
- Subtitle: All conversations

When a channel is selected:
- Title: WhatsApp / Email / Instagram / etc.
- Subtitle: “3 conversations” or “1 conversation”
- Use the filtered row count, not total messages.

Design style:
- Clean white rows
- Good spacing
- Subtle borders
- No heavy colors
- Channel badge small and elegant
- Do not make the row taller than necessary
- Keep mobile readable
- Preserve click-to-open behavior
- Preserve selected row state
- Preserve escalation badges if present
- Preserve timestamp/star behavior
