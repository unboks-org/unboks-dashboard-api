Issue:
Each message bubble has a Translate language selector.
When the operator chooses a language on one message, the other Translate controls still stay on the old default language.

Bad UX:
The operator has to change the language again for every single message.

Required behavior:
1. Translation target language should be shared globally across the conversation view.
2. Default target language is English.
3. If operator changes language to Dutch on one message, all Translate controls immediately show Dutch.
4. If operator changes language to Spanish on one message, all Translate controls immediately show Spanish.
5. The next message translated should use the currently selected global language.
6. The selected language should persist during navigation in the dashboard.
7. Prefer persisting selected language in localStorage so refresh keeps the preference.
8. Suggested localStorage key:
   unboks_translation_target_language
9. Do not make the operator choose language per message unless they intentionally change it.
10. Existing cached translations must remain correct per message and per language.

Important caching rule:
Cache translation by:
messageId + targetLanguage

Example:
- message 123 translated to English
- message 123 translated to Dutch

These are separate cached translations.

If global language changes:
- already shown translation in another language can either remain visible with its correct label, or be hidden until retranslated.
- Preferred behavior:
  If a visible translation does not match the selected language, show a small “Translate to Dutch” state instead of showing the old English translation as if it were Dutch.
- Never label an English translation as Dutch.

Preferred implementation:
Create shared translation language state at the conversation/detail level.

Example:
const [translationLanguage, setTranslationLanguage] = usePersistentTranslationLanguage()

Pass these props into each message bubble:
- translationLanguage
- onTranslationLanguageChange

Do not keep independent language state inside every message bubble.

Language options:
- English
- Dutch
- Spanish
- Papiamento
- Portuguese
- Swedish

UI behavior:
- Every Translate control shows the same selected language.
- Changing the dropdown on any message updates all controls.
- The Translate button sends targetLanguage equal to the shared selected language.
- The translation result label says:
  Translated to Dutch
  Translated to Spanish
  etc.
