BUG:

Fonts are too faint, too small, and background/text contrast is poor.

Required result:
The mobile dashboard must be clearly readable on phone screens.

This must be fixed globally, not only on one screen.

Audit and fix these areas:
- mobile header/top bar
- hamburger menu/drawer
- sidebar/mobile nav labels
- Inbox

Contrast requirements:
- Main text must use a strong readable color, not faint grey.
- Secondary text must still be readable.
- Labels must not be tiny pale grey.
- Button text must be readable.
- Badge/pill text must be readable.
- Input placeholder text must be readable enough, not invisible.
- Active states must be obvious.
- Background and card colors must not be too close to text colors.

Typography requirements:
- Body text on mobile should generally be at least 14px.
- Important row text/customer names/buttons should be 14–16px.
- Page titles/section titles should be 18–22px where appropriate.
- Avoid ultra-light font weights for important text.
- Use stronger font weight for labels and headings.
- Keep line-height comfortable.

Suggested color direction:
Use a premium high-contrast neutral palette:
- Primary text: deep navy/charcoal, for example #111827 / #172033 / #1E293B
- Secondary text: stronger slate, for example #475569 / #4B5563
- Muted text: do not go lighter than roughly #64748B for important UI
- Borders: #D8DEE8 / #E2E8F0
- Background: clean off-white/white
- Active accents: existing premium blue/teal accent, but readable

Do NOT:
- use pale grey text for important info
- use grey-on-grey low contrast
- use pure black everywhere
- use neon colors
- change functionality
- hide information
- make fonts huge and clumsy
- create horizontal overflow
