PHASE 2 — REQUIRED MOBILE FIXES

1. Mobile navigation / hamburger menu

Add a proper mobile navigation pattern.

Required:
- On mobile, show a hamburger/menu button in the top bar.
- Tapping it opens the navigation.
- Navigation can be a slide-in drawer or overlay.
- Drawer must include:
  - Escalations
  - Inbox
  - Channels: WhatsApp, Email
  - Workspace: Appointments, Analytics, Settings
- Drawer must be easy to close:
  - close X
  - tap outside
  - selecting nav item closes drawer
- Desktop sidebar must remain unchanged.
- Mobile nav must not cover content permanently.
- No broken sideways sidebar on mobile.

Visual style:
- premium drawer
- clean spacing
- readable labels
- proper active state
- good contrast
- no clutter

2. Mobile typography

Fix readability.

Minimum targets:
- Body text should not be tiny.
- Main text should be at least 14px where practical.
- Important labels/buttons should be 14–16px.
- Page titles should be readable, around 18–22px depending on layout.
- Avoid faint grey text on light grey backgrounds.
- Increase contrast for secondary text.
- Use consistent font weights.

3. Contrast and colors

Fix poor contrast.

Required:
- Text must be readable against background.
- Secondary text still readable.
- Active nav states visible.
- Badges/pills legible.
- Buttons clearly clickable.
- Do not use ultra-light grey text for important information.
- Keep premium calm palette, but improve contrast.

Do not:
- use pure black everywhere
- use neon colors
- use cheap gradients
- use random colors
- make it look playful

4. Layout and spacing

Fix crooked layouts.

Required:
- No horizontal overflow on mobile.
- No components clipped off-screen.
- Cards align cleanly.
- Content has consistent side padding.
- Lists use clean row spacing.
- Headers do not collide with buttons.
- Footer action bars do not block content.
- Composer remains usable.
- Modals fit within viewport.

5. Inbox and escalation list mobile

Make message lists readable and usable.

Required:
- Rows should stack naturally.
- Customer name, channel, time, status should be readable.
- Preview text should not be crushed.
- Icons/actions should not crowd the row.
- Tap targets should be large enough.
- Active/Archived filters must fit cleanly.
- Channel badges must not wrap awkwardly.

6. Conversation detail mobile

Make the detail panel usable on mobile.

Required:
- Detail view should fit mobile width.
- Header readable and not overcrowded.
- Decision/summary card readable.
- Composer textarea usable.
- Action buttons readable and tappable.
- Send / Resolve / Send & Resolve buttons should stack or wrap cleanly.
- No tiny icon-only controls unless obvious.
- No horizontal scrolling.

7. Settings mobile

Settings must not look broken.

Required:
- Cards stack cleanly.
- Forms are readable.
- Inputs/buttons fit width.
- Danger zone cards, notes, Source of Truth / Company knowledge sections must be readable.
- No tiny labels.

8. Appointments / Analytics mobile

Required:
- Cards/tables must adapt to mobile.
- If tables exist, convert visually to stacked cards on mobile or make overflow controlled and polished.
- No broken columns.

9. Modals and drawers

Required:
- Modals fit mobile screen.
- Padding is reduced appropriately.
- Buttons remain reachable.
- Overlay has good contrast.
- Close buttons visible.

10. Touch targets

Required:
- Buttons and important controls should be around 40–44px minimum height where practical.
- Links/buttons should not be too close together.
- Hamburger menu must be easy to tap.
