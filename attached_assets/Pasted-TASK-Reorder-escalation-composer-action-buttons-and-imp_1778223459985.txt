Current issue:
The escalation composer action buttons currently show in this order:
- Send + resolve
- Send only
- Mark resolved

This order is wrong and visually weak.

Required new order:
1. Send
2. Resolve
3. Send & Resolve

Use these exact labels:
- Send
- Resolve
- Send & Resolve

Meaning:
- Send = send the current reply/guidance only, do not resolve escalation.
- Resolve = mark escalation resolved without sending a reply/guidance.
- Send & Resolve = send the current reply/guidance and mark escalation resolved.

Design goal:
Use Refero to make these action buttons feel premium, calm, and clear.
This is an operator decision area, so it must be visually obvious but not aggressive.

UX priority:
The safest order is:
Send → Resolve → Send & Resolve

Reason:
- Send is the normal action.
- Resolve is separate.
- Send & Resolve is the combined final action.

Button styling:
All three actions should look like part of the same action group.
Do not make them feel like random mismatched links.

Suggested visual hierarchy:
1. Send
   - primary or strong secondary action
   - enabled when there is text/guidance to send

2. Resolve
   - secondary/outline action
   - should be clearly clickable
   - not a tiny text link

3. Send & Resolve
   - strongest/final action
   - use premium accent color
   - disabled when there is no text/guidance to send, if that is the current logic
   - should visually communicate this is the combined final action

Important:
Do not use ugly grey disabled pills unless the action is actually disabled.
Do not use tiny text-only links for important actions.
Do not use confusing labels like “Send only.”
