Issue:
Email detail currently renders as one ugly raw text blob.

Example:
“we good ? what time can we meet this week to activate my service ? Calvin -- *Calvin Adamus* Co-Founder GAIMIN Group *Email:* Calvin@gaimin.io Confidentiality Warning: ...”

This is not acceptable.
It feels like a dump of words, not a premium email/message interface.

Goal:
Create a clean, premium email detail layout.

Use Refero/Repro:
Use real email/message detail references before redesigning.
References should include:
- Gmail email detail
- Front email client
- Superhuman
- Intercom conversation detail
- Zendesk message detail
- Linear/Stripe clean panel layout

If Refero/Repro is not connected, stop and report clearly before editing.

Required behavior:
1. Email detail pane must not show one raw text dump.
2. It should display a clean email-style message.
3. Extract and separate:
   - main message body
   - sender signature if present
   - confidentiality/disclaimer footer if present
4. Signature/disclaimer should not dominate the message.
5. Long legal confidentiality text should be collapsed by default or visually de-emphasized.
6. Message body should be the focus.
7. Preserve the full content somewhere accessible, but do not make the UI ugly.
8. Do not alter WhatsApp message display unless shared component requires minor safe polish.
9. Do not change backend/API.

Email layout:
Header area:
- Sender name or email
- Channel badge: Email
- Timestamp
- Action icons/buttons:
  - Reply
  - Forward
  - Delete

Message card:
- Main body text, readable and clean
- Proper line breaks
- No raw markdown stars if possible
- No giant single paragraph if signature exists

Signature:
If detected, show below main body in a lighter section:
- Smaller text
- Muted color
- Optional label: Signature
- Do not let it overpower the message

Confidentiality warning/disclaimer:
If detected, show collapsed by default:
- Label: Confidentiality notice
- Button: Show / Hide
- Muted styling
- Do not show full legal text by default unless expanded

Parsing rules:
Detect common separators:
- "--"
- "Kind regards"
- "Best regards"
- "Regards"
- "Met vriendelijke groet"
- "Confidentiality Warning:"
- "This message and any attachments"
- "If you are not the intended recipient"
- "written with Chatgpt"

If a confidentiality warning starts, move everything from that point into a collapsible notice.

If a signature starts, move signature into signature block.

If parsing fails, still render the text with good spacing and line breaks.

Markdown-ish cleanup:
- Convert `*Calvin Adamus*` to bold if safe, or remove the literal asterisks.
- Convert `*Email:*` to bold label if safe, or clean to `Email:`.
- Do not display raw markdown markers unless necessary.

Visual style:

- White background
- Clean message card
- Good spacing
- Main message text 14 to 15px, readable
- Secondary/signature text muted but readable
- No huge grey blob

- No heavy shadows
