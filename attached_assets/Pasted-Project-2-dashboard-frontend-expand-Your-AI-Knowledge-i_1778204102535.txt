Product requirement:
Settings currently has a section called “Your AI knowledge”.

This section must become the customer-facing knowledge hub for everything the AI can use when replying to customers.

The customer must be able to:
1. Add manual knowledge updates.
2. Upload documents and images.
3. Connect cloud storage sources like Google Drive and OneDrive.

This is the customer-facing version of Source of Truth.
Do not call it “SOT” in the UI.
Use “Your AI knowledge”.

Goal:
Make it clear that this is where the customer gives the AI business knowledge.

Section title:
Your AI knowledge

Subtitle:
What your AI already knows about your business.

Better expanded description:
Add business information, files, policies, offers, and cloud folders your AI can use when answering customers.

Required layout:
Inside “Your AI knowledge”, create a premium knowledge hub with 3 main cards/areas:

1. Add update
Purpose:
Quickly add text information the AI should know.

Examples:
- “We are closed on Christmas Day.”
- “Valentine’s Day offer: 10% discount.”
- “We are fully booked this Saturday.”
- “For pricing questions, ask for number of channels first.”

UI:
- Textarea
- Category selector:
  General
  Offer
  Holiday
  Hours
  Pricing
  Policy
  Other
- Optional active dates:
  Start date
  End date
- Save button

This should connect to the existing Your Info Updates logic if already present.

2. Upload knowledge files
Purpose:
Upload files the AI can use as knowledge.

Supported file types in UI:
- PDF
- DOC / DOCX
- TXT
- CSV
- XLS / XLSX
- PNG
- JPG / JPEG
- WebP

UI:
- Upload area / drag and drop zone
- Button: Upload files
- Helper:
  Upload documents, menus, price lists, FAQs, screenshots, and policies.
- Show file list:
  filename
  type
  upload date
  status
  remove button

Statuses:
- Pending
- Processing
- Ready
- Failed

Important:
If backend upload is not connected yet, show calm placeholder:
“File upload will be connected by the Unboks team.”
Do not pretend the file is available to the AI if backend did not confirm.

3. Connect cloud storage
Purpose:
Connect external cloud folders where business knowledge lives.

Cloud providers:
- Google Drive
- OneDrive
- Dropbox
- SharePoint
- Box

UI:
Show provider cards/buttons:
- Google Drive
- OneDrive
- Dropbox
- SharePoint
- Box

Each provider should show:
- Connect button if not connected
- Connected state if connected
- Choose folder button if connected
- Sync status
- Last synced date if available

Helper copy:
Connect folders with documents, price lists, FAQs, menus, and policies your AI should use.

If backend is not connected:
Clicking Connect should show:
“Cloud connections will be connected by the Unboks team.”

Do not fake connection success.
