Issue:
After a task is saved, there is no way to edit it.
Users need to correct or update task text after saving.

Task:
Add an Edit action to saved task cards.

Required behavior:
1. Each task card should have an Edit button/icon.
2. Edit should work for local pending tasks immediately.
3. When clicked, the task enters edit mode.
4. Edit mode should allow editing:
   - task text/body
   - assigned person if feasible
   - attachments if already supported and easy
5. Save button updates the task.
6. Cancel button exits edit mode without saving changes.
7. Updated task remains persisted after refresh for local pending tasks.
8. If backend task editing endpoint does not exist yet, do not pretend shared backend tasks are editable.
9. For backend/shared tasks, either:
   - disable Edit with tooltip/copy “Editing shared tasks will be available when the backend supports it.”
   or
   - use PATCH /api/unboks/tasks/:id if the existing API client already supports body updates.
10. Do not break Mark done / Reopen.
11. Do not break local pending sync logic.
12. Keep the UI premium and simple.

Preferred UI:
- Add a small “Edit” button in the task card header or footer.
- In edit mode:
  - show textarea with current task text
  - show To selector if assignment editing is simple
  - buttons:
    - Save changes
    - Cancel
- Keep card layout clean.
- Do not open a giant modal unless current task design already uses modals.
- Mobile must be comfortable.

Local pending task behavior:
- Editing a local pending task updates the localStorage copy.
- syncStatus should remain pending.
- updatedAt should update.
- If the task is done, allow edit but keep done status unchanged.

Backend/shared task behavior:
- If editing backend tasks is not supported, show edit disabled or a calm note.
- Do not silently fail.
- Do not fake successful backend edits.
