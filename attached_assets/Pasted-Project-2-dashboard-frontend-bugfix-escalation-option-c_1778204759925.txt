BUG:
The option chips in the Escalation Reason panel are not clickable.

Current examples:
- Confirm Thursday at 09:00
- Suggest another time
- Ask Marina to collect more availability
- Switch to human takeover
- Mark resolved

They look like buttons/chips, but clicking them does nothing.

This is bad UX.

Required behavior:
Option chips must be real actions.

Soft escalation behavior:
The chips should help the operator quickly reply to Marina.

When clicking a guidance option, fill the “Reply to Marina” composer with clear guidance text.

Examples:

Chip:
Confirm Thursday at 09:00

Composer text:
Please confirm Thursday at 09:00 with Calvin.

Chip:
Suggest another time

Composer text:
Please let Calvin know that Thursday at 09:00 does not work and ask him to suggest another time.

Chip:
Ask Marina to collect more availability

Composer text:
Please ask Calvin to send 2 or 3 other times that work for him this week.

Hard escalation behavior:
If current mode is hard, chips should fill the “Reply to customer” composer with customer-facing text.

Examples:

Chip:
Confirm Thursday at 09:00

Composer text:
Thursday at 09:00 works for us. We’ll confirm the activation meeting for that time.

Chip:
Suggest another time

Composer text:
Thursday at 09:00 does not work for us. Could you send 2 or 3 other times that work for you this week?

Chip:
Ask for more availability

Composer text:
Could you send 2 or 3 times that work for you this week? Then we can confirm the activation call.

- It should use the existing mode switch logic.
- It should not only fill text.
- After switching, composer should become “Reply to customer”.

2. Mark resolved
- Should trigger the existing mark resolved behavior if available.
- If backend endpoint is not connected, show current safe placeholder/error behavior.
- Do not silently fail.

1. Chips must be buttons, not static spans.

3. They must be keyboard accessible.
