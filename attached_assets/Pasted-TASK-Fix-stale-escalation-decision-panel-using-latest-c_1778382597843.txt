BUG:
The Escalation detail panel shows stale decision information.

Current example:
Decision card says:
“Calvin suggested tomorrow evening 17:00 or Monday morning 11:00.”

But the latest customer message says:
“i changed my mind, i wanna change it to friday 12:00”

The latest customer message is visible, but the Decision Needed / What Happened / Suggested Next Step panel did not update.

This is wrong.

The decision card must reflect the latest relevant customer information, especially when the customer changes availability, request, booking details, or appointment preference.

CORE RULE:
The Decision Needed panel must be based on the latest conversation state, not only the original escalation summary.

If backend escalation summary is stale, frontend must defensively derive a fresher decision summary from the current conversation detail.

REQUIRED BEHAVIOR:
1. When an active escalation is open, inspect the latest messages in the conversation.
2. Prefer backend structured escalation summary only if it matches the latest customer state.
3. If the latest inbound customer message contradicts or updates the old summary, show a frontend-derived “latest update” decision summary.
4. The “Latest customer message” section already shows the new message. The main Decision Needed panel must also reflect it.

Example:

“Customer offered tomorrow 17:00 or Monday 11:00.”

Latest customer message:
“i changed my mind, i wanna change it to friday 12:00”

Correct decision panel:

Options:

- Suggest another time
- Ask for more availability
- Switch to human takeover

Do not keep showing tomorrow 17:00 or Monday 11:00 as the main decision if the latest customer message changed it.
