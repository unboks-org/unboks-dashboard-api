Bug:
Email conversations appear in the Email list.
But when clicking an email conversation, the right-side detail pane opens but does not show the actual email/message body.
It shows the contact/header area, then empty/loading placeholder bubbles.

WhatsApp detail may work, but Email detail is broken.

Important:
The Python backend uses the `phone` field as the conversation identifier.
For email conversations, `phone` can contain special characters, for example:
email::subj:workspace-noreply@google.com:boost productivity...

Frontend must safely fetch details for these email conversation IDs.

Required behavior:
1. Clicking an Email row must show the full message history/body in the detail pane.
2. Email detail request must use the exact backend-provided conversation identifier.
3. The identifier must be URL encoded with encodeURIComponent before building the URL.
4. Do not double-encode.
5. WhatsApp detail must still work.
6. Message body must render from whatever field the Python backend returns:
   - text
   - content
   - body
   - message
7. Timestamp must render formatted, not raw ISO.
8. If the detail API fails, show a readable error in the detail pane, not empty placeholder bubbles.
9. Do not show fake/mock detail messages.
10. Do not change backend/API.
