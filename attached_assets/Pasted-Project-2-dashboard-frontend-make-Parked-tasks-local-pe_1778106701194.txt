Product decision:
Parking a task is a personal/local action for the current user.
It is not a shared/backend status yet.

Meaning:
- If Calvin parks a task, it disappears from Calvin’s Open list.
- It appears in Calvin’s Parked tab.
- It should not disappear from Jr’s own view unless Jr also parks it.
- Parked state is local to the current browser/user until backend supports per-user task states.
- Do not send parked status to backend yet.
- Do not show red sync errors when parking.

Current issue:
Clicking Park shows:
“Parking shared tasks will be available when sync supports it.”

This is bad UX.
Parking should work locally now.

Required behavior:
1. Clicking Park saves a local parked state for that task.
2. Parked task disappears from Open.
3. Parked task appears in Parked.
4. Parked task appears in All with a Parked badge.
5. Parked state persists after refresh.
6. Move to open removes the local parked state.
7. Mark done should still work.
8. Done should override parked where appropriate:
   - if task is marked done, it appears in Done
   - if reopened, local parked state can remain or clear; prefer clear to Open
9. Do not call backend PATCH with status="parked" for shared/backend tasks.
10. Do not show red error for Park.
11. If a task is local pending, parking can update the local pending task directly.
12. If a task is shared/backend, parking should store only a local hidden/parked override.
13. Jr should not be affected by Calvin parking a task, because it is local/personal.
14. Calvin should not see parked tasks in Open.
15. Parked tasks should be clearly labelled “Parked”.

Storage:
Use localStorage for local parked overrides.

Suggested localStorage key:
unboks_parked_task_ids

If task ids are not stable, use the best stable key available:
- backend id if available
- localId if local pending
- avoid using task text alone if possible

Recommended model:
localParkedTaskIds: string[]

Filtering logic:
- Open tab:
  show tasks with status open AND not locally parked
- Parked tab:
  show tasks whose id is in localParkedTaskIds OR status === "parked" for local pending tasks
- Done tab:
  show done tasks
- All tab:
  show all tasks, with parked badge if locally parked
